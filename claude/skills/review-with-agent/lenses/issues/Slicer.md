<directive>
Evaluate the issue set based on vertical slicing and independent deliverability.

Questions to ask:
1. Do the issues represent "tracer bullet" vertical slices (touching DB, Backend, and Frontend as needed for a single user value), rather than horizontal layer-cake slices (e.g., "build all DB tables" then "build all APIs")?
2. Can each completed issue (or small group of issues) be verified end-to-end?
3. Is the project broken down in a way that allows early feedback on integration?
4. Do the slices align with the phased rollout described in the PRD/plan?
