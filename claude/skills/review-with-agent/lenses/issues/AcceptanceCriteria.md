<role>Acceptance Criteria</role>
<directive>
Evaluate the issue set based on the clarity, testability, and completeness of the Acceptance Criteria.

Questions to ask:
1. Does each issue have a clear `## Acceptance Criteria` section?
2. Are the criteria binary (can be demonstrably verified as pass/fail)?
3. Do the criteria cover the functional requirements defined in the PRD/plan?
4. Are edge cases and error handling accounted for in the ACs?
5. Will an agent grabbing this issue know *exactly* when they are done?
</directive>
