# Phase 6: REPORT

Generate the final research report from collected and validated data.

## Inputs
- Read `state.json` (type, topic, auto flag)
- Read all JSON files from `data/`
- Read `validation.json` and `critique.json` if they exist

## Step 1: Generate Report Skeleton
Run the report generator to create the initial skeleton:
```bash
python3 ~/.claude/skills/research/scripts/generate_report.py <type> "<topic>" data/ ~/.claude/skills/research/templates/
```

This produces a skeleton with `[Generated by LLM orchestrator during Phase 6]` placeholders.

## Step 2: Fill Placeholders

Replace each placeholder section with substantive content based on the research data:

### Executive Summary (200-400 words)
Synthesize the key findings. Include:
- What was researched and why
- Top 3-5 findings
- Overall recommendation or conclusion

### For Comparison Mode:
- **Comparison Table**: Already generated by the script from field data
- **Per-Dimension Analysis**: For each comparison dimension, write analysis based on the entity JSON data. Reference specific data points. Cite sources with `[N]` format.
- **Recommendation**: Based on findings, state which entity is best for which use case

For the TOC display fields: unless `--auto`, ask the user which fields to show in the summary table via `AskUserQuestion`. If `--auto`, auto-select fields with `detail_level: "brief"`.

### For Evaluation Mode:
- **Scorecard**: Score each dimension 0-100 based on the research data. Write a one-line assessment per dimension.
- **Per-Dimension Detail**: Write detailed analysis for each dimension
- **Risk Assessment**: Summarize integration risks
- **Verdict**: State clearly: **Recommend** (score >= 70), **Conditional** (50-69), or **Not Recommended** (< 50). Explain why.
- **Alternatives**: List and briefly compare alternatives found

### For Deep Mode:
- **Introduction**: Set context for the research topic
- **Section Content**: Write detailed sections based on each search angle's findings
- **Synthesis**: Connect insights across sections, identify patterns
- **Limitations**: What this research didn't cover
- **Conclusion**: Key takeaways
- **Methodology**: Describe the search process used

### Bibliography
Compile all sources from data JSON files. Format as:
```
[N] Title. URL
```
Ensure every `[N]` citation in the text has a matching bibliography entry.

## Step 3: Write Report
Write the completed report to `report.md` in the output directory.

## Step 4: Write Sources
Write all source data to `sources.json` in the output directory for future reference.

## Step 5: Validate Report
Run structural validation:
```bash
python3 ~/.claude/skills/research/scripts/validate_report.py report.md
```

If validation fails:
1. Read the error messages
2. Fix the issues in the report
3. Re-run validation
4. Maximum 3 fix cycles

## Step 6: Report Completion
Tell the user:
- Report path: `{output_dir}/report.md`
- Word count
- Number of sources
- Validation result (PASS/FAIL)
