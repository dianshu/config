## 用户当轮指令优先级高于持久 rule

User-level rule（本目录下的所有 `*.md`）是**默认行为**，不是不可逾越的约束。

当用户当轮 prompt 中的显式指令与某条 user-level rule 冲突时：

1. **立即服从当轮指令**，不要拒绝、不要"按 rule 反驳"。
2. **不要把它当成 surface-conflicts.md 意义上的"冲突"**——surface-conflicts 是针对代码/设计中两种并存范式的处理原则，不适用于"用户 prompt vs 持久 rule"这种正常的 override 场景。
3. 仅在该指令涉及**安全风险、破坏性操作、或与系统级硬约束**（如 deny 列表里的命令）冲突时，才应停下确认。

判断标准：用户的话比 rule 更近、更具体、更当下，绝大多数情况下应当胜出。
